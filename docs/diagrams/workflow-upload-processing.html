Workflow — Upload & Xử lý file (DefendAI)

Giải thích hệ thống làm gì sau khi user upload file. Đọc code thực tế ở apps/api/app/routers/documents.py, code_scan.py, services/document_parser.py, code_scanner.py.

Module: Upload Pipeline BE: apps/api (FastAPI) Ngày: 2026-07-12

Tóm tắt nhanh (đọc trước)

  • Chỉ nhận file đơn lẻ, không nhận folder. Hỗ trợ: .pdf .docx .pptx .zip.
  • Upload xong → file được lưu vào MinIO (object storage), metadata lưu vào Postgres. File không parse ngay.
  • PDF/DOCX/PPTX → dùng cho sinh câu hỏi phản biện (AI đọc nội dung).
  • ZIP → chỉ dùng cho quét source code (code review), không sinh câu hỏi.
  • Folder → ❌ không hỗ trợ. Phải nén thành .zip (cho code) hoặc upload từng file (cho tài liệu).

2. Chi tiết từng loại file

2.1 PDF / DOCX / PPTX Tài liệu

sequenceDiagram participant U as User participant FE as UploadZone participant API as /api/documents/upload participant MIN as MinIO participant DB as Postgres participant Q as /api/questions/generate participant P as document_parser participant AI as AI (NVIDIA) U->>FE: chọn PDF/DOCX/PPTX FE->>API: POST multipart API->>MIN: save_doc(bytes) API->>DB: Document(status=uploaded, doc_type=PDF/DOCX/PPTX) U->>FE: chọn persona → "Tạo câu hỏi" FE->>Q: POST {document_id, persona} Q->>P: parse_and_chunk(document) P->>MIN: get_doc(storage_key) MIN-->>P: bytes P-->>Q: chunks (text) Q->>AI: Map-Reduce: nhóm 3 chunk → sinh câu hỏi AI-->>Q: 10 questions + hint (JSON) Q->>DB: Assessment(status=completed) Q-->>FE: questions FE->>U: hiển thị /questions
  • Parse text: document_parser.py — PDF qua PyPDF2, DOCX qua python-docx, PPTX qua python-pptx.
  • Chunk ~4000 ký tự/chunk, overlap 600 (giữ ngữ cảnh).
  • AI Map-Reduce: chia chunks theo nhóm 3 → gọi song song → tổng hợp 10 câu. Thiếu thì bù bằng heuristic.

2.2 ZIP Source code

sequenceDiagram participant U as User participant FE as UploadZone participant API as /api/documents/upload participant MIN as MinIO participant DB as Postgres participant SC as /api/code/scan participant Z as code_scanner participant AI as AI (NVIDIA) U->>FE: chọn ZIP (source code) FE->>API: POST multipart API->>MIN: save_doc(bytes) API->>DB: Document(status=uploaded, doc_type=ZIP) U->>FE: "Bắt đầu phân tích" FE->>SC: POST {document_id} SC->>Z: analyze_code_document(document) Z->>MIN: get_doc(storage_key) Z->>Z: extract_code_files (giải nén, lọc .git/node_modules) Z->>AI: review code → JSON issues AI-->>Z: summary, pass_rate, issues[] Z->>DB: CodeAnalysis(status=completed) SC-->>FE: issues + pass_rate FE->>U: hiển thị /code-review
  • Giải nén ZIP → lọc folder rác (.git, node_modules, dist...), chỉ lấy file code (.py/.ts/.js/...).
  • Giới hạn: tối đa 500 file trong zip, 50 file scan, 20MB giải nén.
  • AI trả issues có file + line thật. Nếu AI lỗi → fallback heuristic (regex bắt secret/TODO/exec) — là sự thật deterministic.
  • ZIP không dùng để sinh câu hỏi (question_generator từ chối doc_type=ZIP).

  • Backend ALLOWED_EXTENSIONS = {pdf, docx, pptx, zip} — folder không có extension → 400 File type not supported.
  • Muốn upload thư mục code → nén thành .zip rồi upload (hệ thống tự giải nén).
  • Muốn upload nhiều tài liệu → upload từng file, hoặc gom vào 1 PDF/PPTX.
