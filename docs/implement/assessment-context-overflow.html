Vấn đề: Upload SV vượt context window LLM (>200K)

1. Vấn đề (Issue)

  • code_scanner.build_prompt() giới hạn MAX_TOTAL_CHARS = 200000. Codebase sinh viên vượt mức → truncate im lặng → LLM chỉ thấy 1 phần → assessment sai, không báo lỗi.
  • Đồ án SV lớn (full-stack, monorepo, nhiều file) dễ vượt 200K.
  • Hướng RAG (retrieve chunk) giải quyết scale NHƯNG mất holistic view → giảm accuracy khi đánh giá tổng thể.

4. Quyết định kiến trúc

  • Full-context = default (accuracy cao nhất cho đồ án nhỏ).
  • Multi-pass = fallback cho đồ án lớn (xử lý được mà không cần embedding service).
  • RAG / embedding = defer đến khi repo cực lớn (>300K).
  • document_chunks luôn populate → sẵn sàng cho citation + RAG sau.
