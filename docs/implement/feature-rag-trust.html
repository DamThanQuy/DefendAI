Plan Implement — Trusted-RAG: AI đọc source & trả lời có căn cứ

Dự án DefendAI · chức năng: AI đọc codebase (RAG) và mọi câu trả lời / gợi ý / đánh giá sinh ra đều phải được tin cậy — tức có nguồn trích dẫn và qua bước kiểm chứng (verification), không bịa.

Module: Trusted-RAG / AI Trust Layer BE: apps/api (FastAPI) Ngày: 2026-07-12 Trạng thái: Plan

Mục tiêu

  • AI đọc được source codebase (ZIP đã upload) thông qua RAG: chunk → embed → retrieve relevant chunks khi sinh câu hỏi / review.
  • Mọi output của AI (question, hint, suggestion, summary) phải gắn provenance: sources: [{file, line, chunk_id, snippet}] để người dùng tự verify → triệt tiêu ảo giác.
  • Thêm verification pass: sinh xong, hỏi lại AI "claim này có được chứng minh bởi source đã cite không?" → đánh dấu verified / confidence, gạt bỏ cái unsupported.
  • Phủ các trường hợp trust khác (xem §6): heuristic là sự thật deterministic, ranh giới in-source vs training-knowledge, cross-consistency slide↔code, abstention khi không có nguồn.

2. Luồng Trusted-RAG

flowchart LR
    A[Upload doc] --> B[Embedding service: chunk → vector]
    B --> C[(document_chunks)]
    D[Câu hỏi user] --> E[Retriever: top-k chunk relevant]
    C --> E
    E --> F[AI sinh answer + sources]
    F --> G[verify → confidence/verified]
Upload (doc/zip) → Ingest: chunk + embed → Lưu pgvector (chunk + embedding + meta) → Generate: retrieve top-k relevant → AI sinh answer + cite sources → Verify pass (claim vs source) → Output: answer + sources + verified

3. Các bước implement

Bước 1 — Embedding & vector store Cần

Thêm app/services/rag/embeddings.py + app/services/rag/store.py

  • Embedding qua NVIDIA/Google embedding model (dùng AIGateway hoặc gọi trực tiếp). Cache theo hash chunk để không embed lại.
  • Lưu vào Postgres + pgvector (Supabase đã hỗ trợ, không thêm infra). Bảng document_chunks(id, document_id, chunk_index, content, embedding vector, meta jsonb).
  • Local dev: defendai.db là SQLite → không có pgvector. Dùng BM25/keyword retrieval làm fallback khi không bật extension. Hosted (Supabase) dùng pgvector.
// store.py (SQLAlchemy async)
class DocumentChunk(Base):
    __tablename__ = "document_chunks"
    id = mapped_column(BigInteger, primary_key=True)
    document_id = mapped_column(Integer, index=True)
    chunk_index = mapped_column(Integer)
    content = mapped_column(Text)
    embedding = mapped_column(Vector(1024))   // tuỳ dim model
    meta = mapped_column(JSONB)               // {file, line, doc_type}

Bước 2 — Retriever Cần

File app/services/rag/retriever.py

  • retrieve(document_id, query, k=5) → top-k chunk theo cosine similarity (pgvector) hoặc BM25.
  • Trả về list {chunk_id, content, meta}. Deterministic: cùng input → cùng citation (quan trọng cho reproducible + caching assessment).
  • Gộp cả source code: với ZIP, retrieve từ chunks đã extract (code_scanner.extract_code_files) → AI có thể đọc cả doc lẫn code khi sinh câu hỏi.

Bước 3 — Generate với citation Cần

Sửa routers/questions.py + schemas/assessment.py

  • Thay vì nhét toàn bộ chunk, retrieve top-k rồi build prompt yêu cầu AI phải cite chunk hỗ trợ mỗi claim.
  • Mở rộng schema AssessmentQuestion: thêm sources: QuestionSource[], confidence: float, verified: bool.
  • QuestionSource { file?: string; line?: int; chunk_id: int; snippet: string }.
// system prompt thêm:
"Mỗi câu hỏi và hint PHẢI đi kèm danh sách sources —
chỉ những chunk_id thực sự chứng minh claim đó.
Nếu không có source, đặt confidence thấp và verified=false."
// schema
AssessmentQuestion {
  id, question, hint, difficulty, persona,
  sources: [{ chunk_id, snippet, file?, line? }],
  confidence: float,   // 0..1
  verified: bool
}

Bước 4 — Verification pass Cần

File app/services/rag/trust.py

  • Sau khi AI sinh answer, gọi 1 lần nữa (dùng model worker/nhỏ cho rẻ): "Claim X có được chứng minh bởi các source đã cite không? (yes/no + lý do)".
  • Set verified và hiệu chỉnh confidence. Claim unsupported → verified=false, FE hiển thị badge "chưa xác minh".
  • Gate by persona: chỉ chạy verify với strict / code-review để tiết kiệm cost; các persona nhẹ có thể skip.

5. Checklist kiểm thử

□Upload PDF → ingest embed → document_chunks có record, embedding không null. □Sinh 10 câu hỏi → mỗi câu có sources[] trỏ đúng chunk_id, snippet khớp nội dung. □Hint bịa (không có trong doc) → verify pass set verified=false, FE hiện badge "chưa xác minh". □Doc rỗng / scan ảnh → retrieve rỗng → abstain, không sinh hint bịa. □Upload ZIP có hardcode secret → heuristic bắt được, AI không thể hạ severity. □Có cả doc + zip → cross-consistency phát hiện ít nhất 1 mâu thuẫn (nếu có). □Caching: cùng document_id + persona → trả assessment cũ (đã có), không generate lại. □Local dev (SQLite) → BM25 fallback vẫn chạy, không crash vì thiếu pgvector.

7. Rủi ro & lưu ý

  • Cost/latency: verify pass = gấp đôi AI call. Gate bằng persona (strict/code-review) để tiết kiệm.
  • pgvector: cần bật extension trên Supabase (CREATE EXTENSION vector;). Local SQLite → BM25.
  • Context bloat: citation snippet cộng thêm token; giới hạn số sources/question (k=3..5).
  • MinIO drift: doc id 4–8 mất object sau lần reformat ổ đĩa → generate sẽ lỗi NoSuchKey; cần re-upload hoặc cleanup metadata.
  • Dead code: làm xong phải xoá question_generator.py để không ai dùng nhầm đường cũ.
