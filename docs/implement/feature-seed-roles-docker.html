2. Tại sao data không đi theo clone

git clone → code + image (rỗng) → docker compose up → Postgres volume mới (rỗng) → ❌ chưa có roles
  • DB data lưu trong Docker volume postgres_data trên ổ cứng máy bạn → không nằm trong git, không nằm trong image.
  • Khi clone về máy khác, volume là mới → bảng rỗng. Phải chạy migration để populate.
  • May mắn: migration đã chứa seed, nên "chạy migrate" = "có đủ 3 role". Không cần export/import data.

3. Các bước implement (auto-migrate)

1Tạo script entrypoint Mới

File apps/api/entrypoint.sh

  • Script chạy migration rồi mới khởi động uvicorn. Đảm bảo DB sẵn sàng trước khi app nhận request.
#!/usr/bin/env sh
set -e
# Chờ DB (optional, compose đã healthcheck db)
echo "⏳ Running Alembic migrations..."
alembic upgrade head
# Seed 3 tài khoản mẫu (bỏ qua nếu SEED_DEMO=false)
if [ "${SEED_DEMO:-true}" != "false" ]; then
  echo "⏳ Seeding demo users..."
  python seed_users.py
fi
echo "✅ Migrations done. Starting API..."
exec uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload

Lưu ý: đặt quyền thực thi chmod +x entrypoint.sh (hoặc dùng sh entrypoint.sh trong Dockerfile để không phụ thuộc permission trên Windows).

1bSeed 3 tài khoản mẫu (1 role / 1 tài khoản) Mới

File apps/api/seed_users.py (chạy sau migrate, trong entrypoint)

  • Tạo 3 user mẫu, mỗi user gán đúng 1 role tương ứng qua bảng user_roles. Dùng chung 1 mật khẩu demo (bcrypt hash).
  • Password hash phải sinh bằng passlib (bcrypt) — không lưu plaintext. Có thể hash trước ở build time hoặc hash trong script lúc chạy (đã có passlib trong requirements).
  • Username = prefix email. auth_provider = 'email', is_active = 1 (theo schema models/user.py).
  • Idempotent: INSERT ... ON CONFLICT (email) DO NOTHING, role lấy theo name đã seed ở Bước 1.
# apps/api/seed_users.py (chạy sync qua psycopg)
import os, psycopg
from passlib.context import CryptContext
pwd = CryptContext(schemes=["bcrypt"], deprecated="auto")

DEMO_PW = os.getenv("SEED_DEMO_PASSWORD", "DefendAI@123")
HASH = pwd.hash(DEMO_PW)

SEED = [
    ("admin",   "admin@defendai.dev",   "Admin DefendAI", "admin"),
    ("mentor",  "mentor@defendai.dev",  "Mentor DefendAI", "mentor"),
    ("student", "student@defendai.dev", "Student DefendAI", "student"),
]

dsn = os.getenv("DATABASE_URL").replace("postgresql+asyncpg", "postgresql+psycopg")
with psycopg.connect(dsn) as conn:
    with conn.cursor() as cur:
        for username, email, full_name, role in SEED:
            cur.execute(
                "INSERT INTO users (username, email, full_name, hashed_password, auth_provider, is_active, created_at) "
                "VALUES (%s,%s,%s,%s,'email',1,now()) "
                "ON CONFLICT (email) DO NOTHING "
                "RETURNING id", (username, email, full_name, HASH))
            uid = cur.fetchone()
            if uid:  # chỉ gán role khi user mới được tạo
                cur.execute("SELECT id FROM roles WHERE name=%s", (role,))
                rid = cur.fetchone()[0]
                cur.execute(
                    "INSERT INTO user_roles (user_id, role_id) VALUES (%s,%s) "
                    "ON CONFLICT DO NOTHING", (uid[0], rid))
    conn.commit()
print("✅ Seed users done")
Lưu ý bảo mật: Tài khoản demo chỉ dùng cho dev/local. Không chạy seed này trên prod (hoặc đổi mật khẩu mạnh + biến env SEED_DEMO_PASSWORD). Mật khẩu mặc định DefendAI@123 phải được đổi ở môi trường thật.

3Đảm bảo alembic.ini trỏ đúng Kiểm tra

File apps/api/alembic.ini

  • Compose truyền DATABASE_URL qua env (postgresql+asyncpg://...@db:5432/defense_db).
  • Alembic phải đọc URL từ env, không hardcode localhost. Kiểm tra migrations/env.py lấy settings.DATABASE_URL (async) → wrap bằng sync_engine nếu cần.
# migrations/env.py (mẫu cần có)
from app.core.config import settings
config.set_main_option("sqlalchemy.url", settings.DATABASE_URL)
Lưu ý asyncpg: Alembic chạy sync. Nếu DATABASE_URL dùng asyncpg, cần thay bằng driver sync (psycopg2/psycopg) trong env.py hoặc cài psycopg2-binary vào requirements. Kiểm tra trước khi merge.

4(Tuỳ chọn) Tách migrate ra service riêng Cần quyết định

  • Cách A (đơn giản, khuyên): entrypoint của api tự chạy alembic upgrade head. Đủ cho dev.
  • Cách B (sạch hơn): thêm service api-migrate chỉ chạy migrate xong exit, api depends_on nó. Tránh race condition nếu scale api nhiều replica.
# docker-compose.yml (Cách B)
  api-migrate:
    build: ./apps/api
    command: ["sh", "-c", "alembic upgrade head"]
    env_file: [.env]
    environment:
      DATABASE_URL: postgresql+asyncpg://postgres:postgres@db:5432/defense_db
    depends_on:
      db: { condition: service_healthy }
    networks: [defense-net]
  api:
    ...
    depends_on: [api-migrate]

8. Rủi ro & lưu ý

  • Race condition: nếu api start trước khi migrate xong → lỗi connect. Entrypoint chạy migrate trước uvicorn giải quyết việc này (Cách A). Scale replica thì dùng Cách B.
  • asyncpg vs Alembic: Alembic chạy sync, DB URL dùng asyncpg. Phải chuyển sang driver sync trong env.py hoặc cài psycopg2-binary.
  • Windows chmod: git trên Windows có thể bỏ quyền thực thi. Dùng sh entrypoint.sh (không cần +x) để an toàn cross-platform.
  • Volume cũ: máy dev đang có data cũ vẫn OK — migrate idempotent, ON CONFLICT DO NOTHING không trùng.
